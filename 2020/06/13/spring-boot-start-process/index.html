Spring Boot 启动流程以及自动加载配置流程梳理
2020-06-13| technology

Spring Boot的精髓在于自动加载默认配置且可修改，降低了开发者使用Spring 的门槛。本文主要梳理Spring Boot是如何自动加载配置到容器中，使程序运行起来。

Spring 是目前非常流行的一款java应用框架，支持非常多的使用场景。繁琐的配置使用户在应用spring的时候提高了使用门槛，而Spring Boot的出现为开发者降低了使用Spring，减少了许多不必要的配置内容，提高了开发效率。Spring Boot的精髓就是在启动中可以将配置自动加载。今天笔者带领大家一起梳理一下Spring Boot在启动过程中是如何将配置信息加载到程序中的。

从启动方法入手

首先我们准备一个spring boot的项目，找到spring boot的启动类，它的结构大致是这样的

我们知道，java程序执行都是从main方法开始的。在main中，执行了SpringApplication.run()方法，这个方法表示SpringApplication的执行入口。在执行run()方法的时候，SpringBoot就已经将所有的配置加载进来了。

@SpringBootApplicationSpring Boot应用标注在某个类上说明这个是SpringBoot的主配置类，SpringBoot应该允许类的main方法启动Spring Boot应用。

我们进入这个注解的定义可以看到：

SpringBootApplication 也是有多个注解构成，这里包含了Spring Boot 的配置注解以及开启自动配置功能。

@SpringBootConfiguration Spring Boot的配置类， 标注在每个类上，表示这是一个Spring Boot的配置类。

@EnableAutoConfiguration 开启自动配置功能。

下面我们看一下这两个注解的细节。

Spring Boot Configuration

@SpringBootConfiguration 表示当前类是一个Spring Boot的配置类，它是一个派生类，作用与**@Configuration**一致，@Configuration是Spring的配置注解。

Enable Auto Configuration

@EnableAutoConfiguration 表示开启自动配置功能，它引用了@AutoConfigurationPackage 注解，@AutoConfigurationPackage 的作用是自动配置包，配置内容就是从这里加载进来的。

它的定义如下：

@Import(AutoConfigurationImportSelector.class) 表示导入哪些组件的选择器，将所有需要导入的组件以全类名的方式返回，之后导入到spring容器中；

这里会导入非常多的自动配置类(xxxAutoConfiguration) 就是给容器中导入这个场景需要的所有组件，并配置好这些组件；免去了手动编写配置类的工作；

程序会在AutoConfigurationImportSelector类中读取配置，并将它们加载到bean容器中去。

Spring Boot在启动的时候，会去META-INF/spring.properties文件中获取EnableAutoConfiguration的指定的值，将这些值通过反射的方式将对应的类导入到容器中，自动配置类生效；

每一个xxxAutoConfiguration类都是容器中的一个组件，都可以加入到容器中，使用他们作为自动配置。这个时候每一个自动配置类进行自动配置。

解析自动配置原理

我们以HttpEncodingAutoConfiguration为例解析自动配置原理。

  • @Configuration 表示这是一个配置类

  • @EnableConfigurationProperties 启动指定的类，这里启动的是ServerProperties类

  • @ConditionalOnWebApplication Web应用条件下整个配置类才会生效

  • @ConditionalOnClass 当前类是否存在，存在才生效

  • @ConditionalOnProperty 判断配置文件中是否存在某个配置，默认生效

自动配置类就是根据当前不同的条件来决定配置类是否生效。 如果配置类生效，将会在容器中添加一个组件

自动配置类有一个有参构造器，将@EnableConfigurationProperties(ServerProperties.class)的配置以参数的形式传递进来。

我们看一下ServerProperties类,类中有多个属性，例如port, address等，它会与server.servlet.encoding进行绑定，生成配置项。

一旦自动配置类生效，会往容器中添加组件，而这些组件的熟悉是从对应的properties类中获取的，每一个属性又和配置文件进行绑定。

Summary

Spring Boot 在启动的时候会自动加载默认配置，会对spring-boot-autoconfiguration包下的METF-INF/spring.factories文件进行扫描，读取可以加载到的类名，通过IOC将符合条件的类加载到容器中去。

每一个自动加载类都会有一个有参构造器，传入spring boot 默认以及用户的配置，这样就实现了自动加载配置的过程。

python 内存管理
2018-03-30|technology

上次面试的时候被问到了Python的内存管理，当时比较懵逼，因此感觉很少人会提到python的内存管理，python作为一个比较高层的编程语言，在想是不是Python的内存管理机制比较高级，不用用户过多的去关心。后来查了一下，其实不然。

变量赋值机制

首先讲一下Python的变量实现机制。Python使用引用的机制来实现变量赋值。我们知道，在编程中，每创建一个变量都会在内存中开辟一个新的空间，而在python中，如果两个变量的值是一样的，那么这两个变量将会指向相同的内存区。python语言是一种解释性的编程语言，它不同于一些传统的编译语言，不是先编译成汇编再编程机器码，而是在运行的过程中，逐句将指令解释成机器码，所以造就了python语言一些特别的地方。

在Python中，只要修改了变量的值，那么变量的引用地址也就会发生了改变。

对于String类型的变量，对于短字符串与整数类型变量一样采用引用存储，长字符串和其他对象可以有多个相同的对象，使用赋值语句创建新的对象。

引用计数

在Python中，每个对象都有指向该对象的引用综述，即引用计数（referebce count）

对象引用对象时，被引用对象的引用计数将会+1

垃圾回收

当某个对象的引用计数减少时，可以使用del关键字将引用删除。

当某个对象的引用数为0时，该对象就要被执行垃圾回收程序。当垃圾回收启动时，Python扫描到这个引用计数为0的对象，就会将它的内存清空。

垃圾回收程序的执行需要达到某个阈值，即分配对象(object allocation)和取消分配对象(object deallocation)的次数的差值。

700即垃圾回收启动的阈值，10表示每10次0代垃圾回收，执行1次1代垃圾回收，最后一个10表示每10次1代垃圾回收，执行1次2代垃圾回收。

也可以通过手动启动垃圾回收。

分代回收

与JVM类似，Python也采用了分代回收的策略。其思想是，存活越久的对象，越不可能成为垃圾被回收。

Python中将所有的对象分为0，1, 2三代，其中每个对象第一次被创建成为0代对象，每经历一次垃圾回收，如果对象依然存活的话，则进入下一代对象。

垃圾回收启动后，会扫描所有的0代对象，如果0代经历一定次数的垃圾回收后，则启动0代和1代的扫描，当1代也经历了一定次数的垃圾回收后，会启动0,1,2代的扫描，对所有的对象进行扫描。

引用环

引用环的出现是由于对象之间相互引用，而如果这些对象都没有用了，但是引用计数并没有减小到0，那么他们也不会被垃圾回收掉，继续占用内存。

假设a对象引用b对象，b对象同时引用a对象，Python首先会复制a和b的引用计数gc_ref_a和gc_ref_b，在遍历所有对象的时候，如果发现a引用了b,那么gc_ref_b就会-1，最后将不为0的对象保留，将为0的对象回收。

Python赋值中地址的改变

Python在赋值改变的时候，其实是引用内存地址在发生变化。

当a=9时以及b=9时它们指向的地址都是一样的。

对于dict, list对象，对象内容发生改变时，引用内存地址并不会发生该表。但当变量赋值到别的地方，那么引用内存地址会发生变化。

在编程赋值中，还有一个概念叫做深拷贝与浅拷贝，Python也是一样的，主要是内存地址是否会发生变化，不再深究。

小结

有时候在看算法思想的时候，感觉很有意思，很多问题处理的很精妙。比如上面提到的引用环是如何打破的，这对于以后在研究其他领域会遇到相似的情况非常具有借鉴意义。

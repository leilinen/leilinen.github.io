linux的软连接与硬链接

在linux中常见的是软连接，而实际上还有一种链接称为硬链接(hard link)，那么他们之间有什么区别那？

inode

在linux存储中，每一个文件都有一个存储文件元信息的区域，叫做inode。inode中存储的内容包含以下内容：

文件的字节数

文件拥有者的User ID

文件的Group ID

文件的读、写、执行权限

文件的时间戳，共有三个：ctime指inode上一次变动的时间，mtime指文件内容上一次变动的时间，atime指文件上一次打开的时间。

链接数，即有多少文件名指向这个inode

文件数据block的位置

可以说，inode表示一个文件的属性。

硬链接

硬链接通过inode的连接产生新的文件名，而不是新的文件。

首先创建spart-all.sh的硬链接，并使用 ll -i 命令查看两个文件的详细信息。其中第1个属性表示文件的inode号码，第3个属性表示文件连接个数。发现这两个文件的inode号码是相同的，且连接个数为2。

如图，/etc/corntab和/root/corntab两个文件指向real块，有些像两个指针指向同一个内存块。

硬链接的优点是 具有安全性 ，也就是说对于指向数据块的文件，删除一个指向文件后数据块并不会被删除。

硬链接的缺点是 不能跨文件系统 和 不能连接到目录 。

特别是由于不能连接到目录的原因，因此它的应用广泛性比较有限。

软链接

软连接(symbolic link) 是创建一个独立的文件，而这个文件会让数据的读取指向它连接的那个文件的文件名。

当源文件删除后，指向文件会打不开源文件的。因此软连接有些像windows里的快捷方式。

总结

从原理上看，软连接和硬链接的区别是连接文件的inode号码与源文件的inode号码是否相同。硬链接原理的理解类似于c中的变量指针指向内存空间。

sh filename 与 ./ filename的区别

在执行文件操作的时候，有的时候要用到sh命令，也有的时候只需要./就可以了，那么他们之间到底有什么区别那？

如果.不在PATH里面，要执行当前目录下的可执行文件，使用全路径：./executable-file

如果要执行一个sh脚本，不管那个脚本有没有可执行权限，都可以使用：sh [file]

这时file是作为参数传给sh的，如果file不在当前目录下，也需要使用全路径。

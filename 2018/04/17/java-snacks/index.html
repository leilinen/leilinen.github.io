InputStream和OutputStream的理解

在看java 输入输出的时候对InputStream和OutputStream 这两个抽象类很好奇，感觉他们的功能与名字正好相反。所以需要一个方法记住他们，网上看到一个比较有意思的理解方法。

1、在java中stream代表一种数据流（源），java io的底层数据元（想像成水龙头）

2、任何有能力产生数据流（源）的javaio对象就可以看作是一个InputStream对象, 既然它能产生出数据，我们就可以将数据取出，java对封装的通用方法就read()方法了－－（出水龙头）

3、任何有能力接收数据源(流)的javaio对象我们就可以看作是一个OutputStream对象, 同样，它能接收数据，我们就可以调用它的write方法，来让它接收数据－－（进水龙头了，呵呵）
